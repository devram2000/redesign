Webpage Redesign: Local Restaurant

What makes a website interface "good" in terms of user experience and visual design?

Intuitively, we can feel when a website interface looks clean and operates well; it is much harder to identify the qualities that "good" interfaces have. This raises the question: is a "good" interface subjective?

The answer to this question should not be as simple as a yes or no answer. Think about it: everyone has different opinions on what innately looks good. Additionally, other factors like age, experience, cultural values, etc. can alter how a person views and interacts with a particular interface. However, there are indeed certain principles that can improve upon overall user experience and visual design, and these can be identified as the defining features of a "good" inteface.

To get some insight on this from a real-world example, I will be evaluating the homepage of a local restaurant called Quitos Seafood Restaurant & Bar (reference the image linked to their website above) for areas of improvement in terms of the usable/accessible aspects of user experience and visual design principles. Ultimately, I will provide a responsive website redesign of the Quitos homepage that will attempt to improve upon the interface.

Usability and Accessibility

Usability Evaluation

Learnability:

Evaluator Description: Will first time users be able to easily traverse and use the interface?

For first time users, the current set up of the Quitos main page is confusing to use for the following reasons:

  • The structure of the webpage is not consistent.
  • There are rows with multiple items that are relatively condensed and rows with only one item surrounded with whitespace for no apparent reason.
  • The "New Fall Hours" on the top of the page differ from the hours later in the page. The dates and times are also not aligned in terms of structure intuitively.
  • The new hours on the top of the page also only have the open time and not the closing time.
  • In the Navigation Bar, the main page of the website (the one currently being evaluated) is not linked. Therefore, the only way to get back to the title screen is by clicking on the top image, which is not signified to be linked in any way.
  • The section in the Navigation Bar for the website contains a Menu and Reservations section when these sections already exist on the main page; this would not be an issue if these links were directed to the same place, but they both link to different sections/webpages.
  • The menus included on the main page and the menus included in the menu section of the Navigation are different (e.g. the Beer and Wine Menu is not included in the page from the Navigation Bar and the Fall Specials Menu is not included in the main page).

Memorability:

Evaluator Description: Will users who have not used the interface in a while recall how to traverse and use the interface with ease?

In terms of Memorability, returning users should not have too much trouble relearning functionality as the website is not too complex and has relatively clear labels.

Efficiency:

Evaluator Description: Will experienced users be able to quickly traverse and use the interface?

Experienced users will mostly return to the website to primarily utilize two functionalities: ordering online or making an online reservation. As of now, these two buttons on the main page are not placed/sized for efficiency.

For instance, the Loyal Lobster Program flyer is almost quadruple the size of the online reservation and online order sections without being 4 times as important. While the reservation button is relatively easy to find, the order online button is in a row of two other functionality boxes (gift cards and hours) and is smaller than both of them, making it much more difficult to efficiently locate and click.

Usability:

In terms of overall usability, this webpage interface is definitely not ideal specifically in the Learnability and Memorability categories. Here are some other miscellaneous and overarching usability issues:
  • The search icon in the Navigation Bar does not work.
  • There are multiple parallex images which are very cool, but they obscure the readability of the website.
  • The first parallax image has text overlayed that inconveniently disappears when you scroll just a little bit.
  • The second parallax image has text overlayed that is hard to read because it does not contrast much from the image.
  • There is effectively no color hierarchy.
  • The background of the menu boxes, items that are supposed to be on the same level, are different shades of blue.
  • The size hierarchy does not reflect importance of the sections.
  • Most of the information in the footer does not need to be there (should be in the About and Contact Us sections of the Navigation Bar).

Accessibility Evaluation

WebAIM WAVE:

I used the WAVE Web Accessibility Evaluation Tool as a check for various accessibility measurements for the Quitos webpage. In summary, there are various places where images are missing alternative text, forms are missing labels, and links are empty; all of which can pose issues in terms of accessibility for the user. There are multiple instances of pdfs being linked, which is a suboptimal way to convey information on a webpage due to navigation difficulties and other accessibility issues. Last, the footer of the website has very low contrast between the text and the background which can make it difficult to read the text.

Screen Reader:

To evaluate accessibility for visually impaired users, I used a Screen Reader to read out the webpage in the audio form. In terms of interpretation, it was not too clear to me what the website was, but that may just be because I am not used to consuming webpage content in the audio form. It seemed like the text jumped around a lot in terms of content category, so that added to the comprehension issues. Also, all of the text that are in the image form were not read out, which means that a substatial portion of the webpage would be missed by visually impaired users.

Google Translate:

To evaluate accessibility for non-english speakers, I used Google Translate to translate the webpage from English to various other languages. Simlar to the issue with the screen reader, the flyers, buttons, and other text in the image form were not translated to the new language. This heavily impacted the overall comprehension of the website because functionalities like "Order Online" and "Join Now" for the Loyal Lobster program were not translated.

Conclusions

While the original interface for Quitos website was usable, it definitely was not ideal! Remember, no interface will ever be universally perfect, but we can always work towards making the interface a convenience instead of an obstacle such that the user can experience the website's functionality as efficiently as possible.

Hopefully, you can now identify and possibly even utilize some of the qualities that make a web interface "good" moving forward!
